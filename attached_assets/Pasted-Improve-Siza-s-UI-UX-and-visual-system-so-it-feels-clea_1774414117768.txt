Improve Siza’s UI/UX and visual system so it feels cleaner, more professional, more modern, and more solid while staying fully aligned with the existing brand guide and design system.

Important constraints:
- Preserve dark-only mode
- Preserve the existing Forge/Siza palette and semantic token model
- Backgrounds must stay in the Forge Deep / Forge Surface family, never pure black
- Forge Violet is the primary accent for CTA, focus, active, selected, and key links only
- Teal and Spark are secondary accents used sparingly for semantic/supportive emphasis
- Keep the visual balance around 80% dark neutrals and 20% accent
- No gradient text
- No glassmorphism
- No glow-heavy generic AI SaaS styling
- No card soup
- No fake trust metrics or fake testimonial energy
- No hype-heavy copy
- No `transition-all`
- Use shared primitives and reusable patterns instead of one-off styling
- The result should feel architectural, technical, calm, premium, and intuitive

Design direction:
- Siza should feel like a serious architectural/developer workspace, not a trendy AI app
- Use structure first: hierarchy should come from layout, typography, spacing, borders, rails, panels, and section headers
- Prefer matte surfaces, restrained shadows, crisp dividers, and consistent workspace shells
- Keep motion subtle and purposeful, using opacity/transform only
- Use precise, confident, technical copy instead of generic startup phrasing

Execute the following implementation tickets:

1. Design tokens and shell cleanup
- Update `apps/web/src/app/globals.css` with stronger semantic tokens for surfaces, borders, focus, motion, and status
- Add workspace shell rules for matte panels, rails, dividers, and restrained shadows
- Remove glow-heavy defaults and ban `transition-all`

2. App frame and metadata alignment
- Update `apps/web/src/app/layout.tsx`
- Replace generic metadata/tagline copy with Siza-aligned language
- Refine global body/background/text rhythm and keep the skip link polished and visible

3. Shared UI primitives expansion
- Improve shared primitives exposed through `packages/ui/src/index.ts`
- Add or standardize reusable patterns like `WorkspaceShell`, `PanelHeader`, `SectionHeader`, `StatusRow`, `SpecRow`, `EmptyState`, and stronger form field primitives
- Prefer reusable semantic patterns over custom one-off screen styling

4. Dashboard shell and top bar redesign
- Refactor `apps/web/src/components/dashboard/DashboardShell.tsx` and `apps/web/src/components/dashboard/TopBar.tsx`
- Make the shell feel infrastructural and consistent across dashboard pages
- Replace the custom notifications dropdown with an accessible shared primitive
- Tighten spacing, breadcrumb hierarchy, search trigger styling, and focus states

5. Dashboard information hierarchy rewrite
